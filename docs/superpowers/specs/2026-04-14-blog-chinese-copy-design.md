# 博客导航中文化与 README 维护手册设计

## 背景

当前博客的整体结构、首页品牌表达和 About 信息卡已经完成，但站点里仍保留一批英文导航和栏目文案，例如 `Home`、`Posts`、`Author`、`Archive` 等。与此同时，`README.md` 虽然已有基础说明，但还没有完全对齐当前仓库的真实维护方式，尤其是分支、GitHub Pages 发布方式和常用入口说明还可以更清晰。

博客的目标是作为长期打磨的个人作品，因此这一轮需要补齐两件事：

1. 让中文读者进入首页和列表页时，第一眼看到的是统一、自然的中文信息架构；
2. 让仓库自身具备“半年后再回来仍能快速上手”的维护手册能力。

## 目标

- 将全站导航和主要栏目文案统一为中文表达；
- 保留 `GitHub`、`Email`、`AI Agent` 这类专有名词的英文写法；
- 将 `README.md` 升级为与当前仓库状态一致的维护说明；
- 保持现有专业、克制的视觉风格，不引入新的花哨样式或复杂交互。

## 非目标

- 不调整站点整体布局结构；
- 不新增页面类型、分类体系或功能模块；
- 不重做主题、不引入新的 Hugo theme；
- 不进行历史文章内容改写。

## 方案

### 1. 统一导航与栏目文案

通过 Hugo 配置和模板文案，将站点核心入口改成中文，包括：

- 导航：`首页 / 文章 / 关于 / 标签`
- 首页眉标与区块：`作者 / 写作 / 关注 / 转型记录`
- 列表页辅助文案：`文章归档 / 页面`
- About 信息卡：`个人信息`

其中 `GitHub`、`Email`、`AI Agent` 保持英文，以维持技术品牌识别度和表达准确性。

### 2. README 升级为维护手册

README 将围绕“这是什么仓库、怎么本地运行、怎么写文章、怎么调整页面、怎么自动发布”来组织，而不是只保留恢复工程时的说明。内容将明确当前真实使用方式：

- 默认发布分支为 `master`
- GitHub Pages 来源为 `GitHub Actions`
- 常见维护入口位于 `content/`、`layouts/`、`assets/`
- 本地预览、构建、测试和推送的常用命令

### 3. 保持小步改动

这一轮只做文案与说明层的优化，尽量复用现有模板和结构，避免为了“中文化”而做不必要的模板重构。这样可以确保改动小、验证快、后续维护成本低。

## 涉及文件

- `hugo.toml`
  - 修改主导航菜单文案
- `layouts/index.html`
  - 修改首页区块眉标与辅助文案
- `layouts/_default/list.html`
  - 修改列表页和 About 页的信息标签文案
- `README.md`
  - 重写为长期维护手册，修正 `main`/`master` 不一致问题

## 验证方式

- 运行 `hugo --minify`，确认站点可成功构建；
- 检查生成后的 `/`、`/about/`、`/posts/`、`/tags/` 页面，确认关键中文文案已生效；
- 检查 `README.md` 中的分支与发布说明是否和当前仓库状态一致。

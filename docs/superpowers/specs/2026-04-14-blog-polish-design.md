# 博客第二轮打磨设计说明

- 日期：2026-04-14
- 仓库：`yinshaojun001.github.io`
- 目标：在已完成 Hugo 重建的基础上，继续提升博客内容表达与首页辨识度。

## 1. 本轮范围

本轮只做三件事：

1. 优化 About 页正式文案
2. 对已迁移旧文章做一轮全量基础清洗
3. 升级首页内容表达与样式细节

## 2. About 页设计

### 2.1 人设定位

About 页明确表达以下身份：

- Java 后端工程师
- 正在转型 AI Agent 工程师
- 成长中的技术写作者

### 2.2 语气

整体基调：

- 专业
- 克制
- 保留真实成长感

不写成简历摘要，也不写成纯个人感想。

### 2.3 结构

About 页分为五段：

1. 开场定位
2. 职业背景（Java / MySQL / 排障 / 性能）
3. 转型方向（AI Agent / Agent Engineering）
4. 写作与博客定位（学习、整理、输出、长期积累）
5. 结尾与交流

## 3. 旧文章基础清洗设计

### 3.1 范围

对现有已迁移旧文章做全量基础清洗。当前范围约 12 篇历史文章。

### 3.2 清洗目标

目标不是逐篇重写，而是统一提升到：

> 结构规整、可读、可继续维护。

### 3.3 优先处理的问题

- 删除无意义分隔符，如 `⸻`
- 去除或替换 Obsidian 痕迹，如 `![[...]]`
- 整理标题与段落结构
- 修复明显错位的列表、表格、代码块
- 为文章补充基础 `summary`
- 根据主题补充一轮基础 `tags`

### 3.4 约束

- 尽量不改原文核心意思
- 优先整理结构，不做大幅重写
- 不在这一轮追求“每篇都精修到出版级”

## 4. 首页升级设计

### 4.1 目标

首页需要实现：

- 仍然偏技术博客
- 但更有个人品牌辨识度
- 第一屏就能说明作者身份与转型方向

### 4.2 风格方向

采用“两者结合”路线：

- 整体克制、耐看、长期可维护
- 细节上增强个人品牌感

### 4.3 升级点

1. Hero 区
   - 更明确表达“Java 后端工程师 → AI Agent 工程师”的身份变化
   - 提升信息层次感

2. 作者卡片
   - 从简单简介升级成更像作者名片的模块

3. 最新文章区
   - 优化卡片层级与阅读入口

4. 主题区
   - 从简单标签展示升级成更像内容入口

### 4.4 本轮不做

- 不引入重动画
- 不改成作品集站
- 不上复杂交互
- 不改变“首页以文章为主”的重心

## 5. 本轮验收标准

完成后应满足：

1. About 页已有正式文案，可直接对外展示
2. 旧文章完成一轮基础清洗，阅读体验明显好于初始迁移结果
3. 首页在不破坏阅读性的前提下，辨识度和完成度提升
4. Hugo 构建仍然通过

## 6. 结论

本轮不是重做结构，而是在现有骨架已成立的前提下，做一轮“内容表达 + 视觉辨识 + 可维护性”的同步升级。

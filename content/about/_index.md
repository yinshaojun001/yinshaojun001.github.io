你好，我是 **尹绍钧**，一名 **Java 后端工程师**，正在转型为 **AI Agent 工程师**。

过去这些年，我的主要积累都在后端开发这一侧，平时接触和思考比较多的内容，也大多围绕 **Java、MySQL、系统排障、性能优化** 这些方向展开。相比单纯把功能做出来，我一直更在意一件事：**问题到底是怎么发生的，我们又是怎么一步步把它分析清楚、解决掉、并沉淀成经验的。** 也因为这个原因，我会习惯把一些问题排查、技术理解和学习过程记录下来。

这两年，我越来越明确地意识到，自己接下来想投入的方向，不只是传统意义上的后端工程，而是更进一步走向 **AI Agent**。所以现在的我，一边仍然扎根在 Java 后端工程师的基础上，一边也在认真地学习和实践 **AI Agent / Agent Engineering** 相关的内容。对我来说，这不是一句“跟风转型”的口号，而是一次真实发生中的技术重心迁移：我希望自己不仅能理解传统系统的构建与稳定性，也能逐步理解模型、工具调用、工作流编排和 Agent 应用在真实场景里的落地方式。

这个博客，也会如实地记录这两条主线。

一条主线是我已经比较熟悉、也愿意持续沉淀的内容，比如 **Java、后端基础、数据库、并发、性能优化、问题排查**；另一条主线，则是我正在进入的新方向，比如 **AI Agent、Agent Engineering，以及一些 AI 应用开发相关的学习、实验和思考**。我并不想把这里写成一个“已经准备好答案”的地方，它更像是一份长期的技术成长记录：有些内容来自经验，有些内容来自正在学习中的理解，有些内容则是我在转型过程中留下的阶段性痕迹。

我写博客，并不是为了把自己包装成某种“很会输出的人”，而是因为我越来越觉得，**学习、整理、输出、再回头修正理解**，本身就是一种适合自己的成长方式。把零散的问题记下来，把模糊的理解写清楚，把踩过的坑复盘出来，时间久了，就会慢慢变成真正属于自己的积累。

所以，如果你看到这里的内容，有时会偏向 Java 后端，有时会偏向 AI Agent，不用觉得它们割裂——这刚好就是我现在真实的状态：**带着已有的工程基础，走进一个新的方向，并尽量把这个过程记录下来。**

如果你也关注 **Java、后端工程、系统排障、性能优化**，或者同样在关注 **AI Agent / Agent Engineering**，欢迎交流。
